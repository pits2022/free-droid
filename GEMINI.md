## Project Overview
**free-droid** (SZABI) builds a fully automated, "Zero-Touch" infrastructure for a sovereign AI robot. The system
bridges a cloud server (Hetzner GEX44) and an edge device (Raspberry Pi 5) over a secure WireGuard VPN, creating a
distributed, hybrid cloud-edge AI environment.

### Core Technologies
- **Infrastructure:** Terraform (Hetzner Cloud, AWS for the S3 state backend).
- **Configuration:** Ansible (WireGuard, Ollama, edge Python runtime).
- **AI Stack:** Ollama serving a self-fine-tuned **3B** model — Qwen 2.5 3B *or* Llama 3.2 3B, **under A/B evaluation**;
  the same model runs in the cloud (higher precision) and on the edge (Q4_K_M, offline fallback). Fine-tuning is done
  on Google Colab (Unsloth QLoRA), never on the server.
- **Networking:** WireGuard VPN, internal subnet `10.0.0.0/24`.
- **Robotics:** Plain Python threading/asyncio control stack over GPIO (`lgpio`). **No ROS 2** in the Hacktivity scope
  (a ROS 2 port is a later-roadmap item).

## Architecture
"Mother-Child" hybrid:
- **mother-001 (Cloud):** Hetzner `gex44` instance in `nbg1`. LLM inference via Ollama.
- **child-001 (Edge):** Raspberry Pi 5. Robot control, local sensing, offline LLM fallback.
- **Connectivity:** private VPN range `10.0.0.0/24` (cloud `10.0.0.1` ↔ edge `10.0.0.2`).

## Directory Structure
- `infra/terraform/`: Hetzner provisioning. `main.tf` (providers + S3 backend), `cloud/` module.
- `infra/ansible/`: configuration. Roles: `wireguard_setup`, `ai_stack`, `edge_robot`.
- `training/`: fine-tuning datasets (Hungarian, Alpaca format), persona benchmark, scoring template.
- `robot/`: (planned) RPi 5 Python control software.
- `docs/free-droid.md`: the full technical spec and phased build checklist.

## Building and Running

### Terraform
```bash
cd infra/terraform
terraform init
terraform apply
```
*Requires `hcloud_token` and AWS S3 access for state.*

### Ansible
Provision the nodes (triggered automatically by Terraform `local-exec`, or manually):
```bash
cd infra/ansible
ansible-playbook -i inventory.ini site.yml
```
*Inventory groups: `[cloud]` = mother-001, `[edge]` = child-001 (generated by Terraform).*

## Development Conventions
- **Zero-Touch:** no manual SSH intervention after the initial image setup.
- **Security:** VPN keys are generated on the fly and exchanged via Ansible facts (`set_fact` / `hostvars`).
- **AI Ethics:** the `training/` data defines a sovereign, honest, Hungarian-only persona grounded in Yotengrit.
- **Network Isolation:** Ollama is reachable only within the VPN subnet `10.0.0.0/24`.
